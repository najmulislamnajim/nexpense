# nexpense

A clean, simple expense tracker with reports and export (CSV / Excel / PDF).

Built with Django (modular monolith) + server-rendered Tailwind CSS templates.

## Architecture

Modular monolith: one Django project (`config`), split into self-contained
feature apps under `apps/`:

- **apps/core** ‚Äî shared base model (`TimeStampedModel`), site-meta context processor.
- **apps/accounts** ‚Äî custom email-based `User` model, auth (login/signup/logout,
  password reset & change, profile).
- **apps/expenses** ‚Äî `Category` and `Expense` models, dashboard, CRUD.
- **apps/reports** ‚Äî filterable report view, CSV/Excel/PDF export.
- **theme** ‚Äî Tailwind CSS v4 (standalone CLI, no Node.js required), generated by `django-tailwind`.

Settings are split by environment: `config/settings/{base,dev,prod}.py`, driven by
a `.env` file (see `.env.example`).

## Getting started

```bash
python -m venv venv
source venv/Scripts/activate        # Windows Git Bash
# venv\Scripts\activate.bat         # Windows cmd
pip install -r requirements/dev.txt

cp .env.example .env                # then edit SECRET_KEY etc.

python manage.py migrate
python manage.py tailwind build     # one-off CSS build (or `tailwind start` to watch)
python manage.py createsuperuser
python manage.py runserver
```

Visit http://127.0.0.1:8000 ‚Äî you'll be redirected to log in / sign up.
Default expense categories are seeded automatically via migration.

### Working on styles

`python manage.py tailwind start` runs the Tailwind CLI in watch mode alongside
`runserver` (run in a separate terminal) and live-reloads the browser via
`django-browser-reload` (dev only).

## Environment variables

See `.env.example`. Key ones:

- `SECRET_KEY`, `DEBUG`, `ALLOWED_HOSTS`
- `DATABASE_URL` (prod only ‚Äî defaults to SQLite if unset)
- `DEFAULT_CURRENCY`, `TIME_ZONE`

## Production

```bash
pip install -r requirements/prod.txt
export DJANGO_SETTINGS_MODULE=config.settings.prod
python manage.py tailwind build
python manage.py collectstatic --noinput
python manage.py migrate
gunicorn config.wsgi:application
```

Static files are served via WhiteNoise. Set `DATABASE_URL` to a Postgres URL
in production (SQLite is fine for small/single-instance deployments).

## Tech stack

- Django 6, custom email-auth user model
- Tailwind CSS v4 (standalone binary, no Node required) + Alpine.js for light interactivity
- openpyxl (Excel export), reportlab (PDF export)
- SQLite (dev) / PostgreSQL-ready (prod) via `dj-database-url`
